# SAE J1939 вҖ” the heritage

ISOBUS did not start from scratch. It took SAE J1939 вҖ” the heavy-duty vehicle network
already proven on trucks and buses вҖ” and built farming on top. Understanding what
came from J1939 (and what ISO 11783 added) explains a lot of otherwise-arbitrary
detail, especially the shape of the identifier and the PDU1/PDU2 split that trips up
newcomers.

## What J1939 contributed

J1939 turns raw CAN bits into a disciplined messaging system:

- **The 29-bit identifier layout** вҖ” priority, parameter group, and source address.
- **PGNs (Parameter Group Numbers)** вҖ” every message is a *named* group of parameters.
- **The transport protocol** вҖ” multi-packet BAM and RTS/CTS (see
  [Part 3](iso11783-datalink-transport.md)).
- **The diagnostic "DM" family** вҖ” DM1 active faults, DM2 history, and the rest, which
  ISO 11783-12 reuses almost verbatim.
- **Address-based addressing** вҖ” which ISO 11783-5 then upgraded to NAME-based claiming.

   в”Ӯ prio   в”Ӯ EDP в”Ӯ PF В· PS   в”Ӯ source address в”Ӯ
   в”Ӯ 3 bits в”Ӯ DP  в”Ӯ (the PGN) в”Ӯ 8 bits         в”Ӯ
   в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

- **Priority** вҖ” lower wins arbitration (safety low, status high).
- **PGN** вҖ” the message identity, assembled from the PF (PDU format) and, sometimes,
  the PS byte.
- **Source address** вҖ” who sent it.

## PDU1 vs PDU2 вҖ” the classic confusion

The single most error-prone J1939 detail, inherited directly by ISOBUS:

```
   PF < 240  вҶ’  PDU1 (point-to-point)
                the PS byte IS the destination address
                e.g. "send this to address 0x26"

   PF вүҘ 240  вҶ’  PDU2 (broadcast)
                the PS byte is part of the PGN itself
                e.g. "wheel speed, to everyone"
```

Get this wrong and you produce a frame that looks valid but routes to the wrong place
вҖ” a destination-specific PGN sent as broadcast, or a broadcast PGN "addressed" to a
node. machbus's `net::Identifier` encodes/decodes this correctly; the failure mode is
worth knowing because it shows up in hand-built frames.

## What ISO 11783 added on top

| J1939 gaveвҖҰ | ISO 11783 addedвҖҰ |
| --- | --- |
| address-based addressing | NAME-based **address claiming** (plug-and-play) |
| BAM + RTS/CTS transport | **Extended TP** for megabyte object pools and files |
| diagnostic DM family | ISOBUS diagnostics wrinkles (a sixth ID field, functionalities) |
| generic vehicle messages | the farming services: VT, TC, FS, SC, implement control, TIM |

In short: J1939 is the grammar; ISO 11783 is the agricultural language written in it.
